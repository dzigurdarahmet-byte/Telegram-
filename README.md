# 🤖 Аналитик ресторана — Telegram-бот

AI-аналитик для ресторана **Вилла Россо** (Сыктывкар). Подключается к iiko (облако + локальный сервер), Яндекс Еде, Google Sheets. Использует **OpenAI** (основной) + **Claude** (резерв) для анализа на естественном языке.

> Просто напишите боту вопрос на русском языке — он поймёт.

## Что умеет

### 📊 Аналитика продаж
| Команда | Описание |
|---------|----------|
| `/today` | Сводка за сегодня (стоп-лист + AI-анализ + график YoY) |

| `/week` | Отчёт за неделю |
| `/month` | Месячный отчёт + график год-к-году |

## Архитектура

```
Telegram ←→ Python-бот ←→ iiko Cloud API (доставка, стоп-лист, меню)
                ↕                    ↕
         OpenAI (основной)     iiko Server API (зал, OLAP-отчёты)

```

## Быстрый старт

### Шаг 1. Получите ключи

| Сервис | Как получить | Обязательно? |
|--------|-------------|--------------|
| Telegram Bot Token | @BotFather → /newbot | ✅ Да |
| iiko API Login | iikoWeb → Настройки → API | ✅ Да |
| OpenAI API Key | platform.openai.com → API Keys | ✅ Да |
| Anthropic API Key | console.anthropic.com → API Keys | ✅ Да (резерв) |
| iiko Server | Локальный сервер iiko (IP + логин) | ⚪ Опционально |
| Яндекс Еда | vendor.yandex.ru → API | ⚪ Опционально |
| Google Sheet ID | URL таблицы зарплат | ⚪ Опционально |

### Шаг 2. Настройте .env

```bash
cp .env.example .env
# Заполните все обязательные переменные
```

### Шаг 3. Запустите

#### Docker (рекомендуется)
```bash
docker compose up -d --build
docker compose logs -f
```

#### Railway (текущий деплой)
1. Fork репозитория
2. Railway → New Project → GitHub Repo
3. Добавьте все переменные из .env.example в Variables
4. Деплой автоматический при push

#### Python напрямую
```bash
python3.11 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## AI: OpenAI + Claude

Бот использует **двойную AI-систему**:
1. **OpenAI** (gpt-4o по умолчанию) — основной движок
2. **Claude** (claude-sonnet-4-20250514) — автоматический резерв при ошибке OpenAI

Модель OpenAI можно сменить через переменную `OPENAI_MODEL` (поддерживаются gpt-4o, gpt-4o-mini, o1, o3-mini).

## Система доступа

1. Новый пользователь нажимает «Запросить доступ»
2. Админы получают уведомление с кнопками ✅/❌
3. После одобрения — пользователь получает полный доступ
4. Для сохранения после перезапуска — добавить ID в `APPROVED_USERS`

## Решение проблем

**«OpenAI ошибка, переключаюсь на Claude»** — нормально, резервный AI работает. Проверьте баланс OpenAI.

**«Таблица не опубликована»** — Google Sheets → Файл → Поделиться → Опубликовать в интернете → CSV.

**OLAP 0 строк** — проверьте `/diag`. Если iiko Server не подключён — данные зала недоступны.

**Бот не отвечает** — проверьте логи Railway. Частая причина: истёк токен Telegram.

**«Организации не найдены»** — проверьте IIKO_API_LOGIN в iikoWeb → API.

**Нет данных за период** — убедитесь что в iiko есть закрытые заказы. Некоторые отчёты требуют iiko Server.
